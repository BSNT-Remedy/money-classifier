Model Card

1. Model Overview

This application uses a Teachable Machine Image Classification Model designed to classify different denominations of Philippine peso bills. The model runs either client-side using TensorFlow.js or server-side via Django, depending on your implementation.

2. Intended Use

The model is intended for educational and demonstration purposes only. It showcases how machine learning can be applied to identify objects based on visual patterns.

  • For classroom demonstrations
  • Student ML projects
  • Prototype-level image classification tasks

Not Intended For:

  • Authenticating real currency
  • Financial or security verification
  • Commercial or industrial deployment

3. Model Architecture

The model was automatically generated by Teachable Machine using a MobileNet backbone. It accepts image inputs and outputs the predicted label along with a confidence score.

4. Training Data

The dataset consists of user-collected images of peso bills across different angles and lighting conditions. These samples are limited and not fully represent all real-world variations.

5. Performance

Accuracy varies depending on the quality of the input:

  • Good lighting → stronger accuracy
  • Clear angle → better predictions
  • Blur or low light → increased misclassification

The analytics dashboard in this application shows:

  • Prediction counts by label
  • Average confidence per label
  • Total predictions made per user

6. Limitations

  • Limited dataset → does not generalize to all real-world bills
  • May misclassify when bills are folded, worn, or partially visible
  • Not resistant to adversarial inputs

7. Ethical Considerations

This model is designed for educational purposes. Users should not rely on it to verify authenticity of currency or make financial decisions.
